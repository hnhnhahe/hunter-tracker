📖 獵人追蹤戰情室 使用說明

📅 每日更新流程
1. 點 user-bar 的 ⚡ 全股價 — 更新所有帳戶即時股價並記錄今日快照
2. 查看戰情室 KPI 和表格，確認各標的狀況
3. 如有停損警告（紅色列）請留意
💼 有交易時的更新流程
1. Schwab → Accounts → Positions → Export → 下載 Individual-Positions-xxxx.csv
2. 展開「🧬 數據源管理」→ ① 初始持股，匯入 Positions CSV
3. Schwab → History → Gain/Loss → Export → 下載 GainLoss_Realized_Details_xxxx.csv
4. ② 已實現損益，匯入 GainLoss CSV
5. Schwab → History → Transactions → Export（選全部時間範圍）→ 下載 Individual_XXX_Transactions_xxxx.csv
6. ③ 交易紀錄，匯入 Transactions CSV（備用，重建時才需要）
7. 點 ⚡ 全股價 更新股價
💾 定期備份建議
• 每週至少備份一次：點「⋯ 更多」→「💾 備份/還原」→「匯出」
• 有重大交易後立即備份
• 備份 JSON 檔請妥善保存，遺失無法恢復
所有指標的計算公式與使用說明
📊 日均產值
公式：累積所有交易的總實現損益 ÷ 累積總持有天數
含意：從開始持有至今，平均每天產生多少獲利
⚠ 持有越久天數越大，分母膨脹導致數值被稀釋，不同持有期的標的無法公平比較
🔄 滾動日均產值（近365天）
公式：最後更新股價日期往回推365天內的已實現損益 ÷ 365
含意：近一年這個標的平均每天幫你賺多少錢，分母固定365天
優點：所有標的基準相同，可公平橫向比較，更能反映當前產線效能
📌 若某標的近一年沒有賣出紀錄，滾動日均產值為 0，代表近一年無實質產值貢獻
📈 產值貢獻%
公式：該標的滾動日均產值 ÷ 全場滾動日均產值總和 × 100%
含意：這個標的佔整體產線產出的比重
判讀：與「持股占比%」對比
🟢 產值貢獻% > 持股占比% → 高效標的，資金使用效率高
🔴 產值貢獻% < 持股占比% → 低效佔資，資金被壓住但沒有對應產出
💹 總日均產值（含未實現）
公式：（累積實現損益 + 目前未實現損益）÷ 累積持有天數
含意：把帳面獲利也算進去，反映含浮盈的整體持倉效率
注意：需要更新股價後才會顯示，隨股價波動
📅 年初至今 YTD%
公式：（目前總資產 - 年初總資產）÷ 年初總資產 × 100%
含意：從今年第一個有記錄的日期至今，整體帳戶的資產成長率
包含：持股市值變化 + 現金 + 已實現損益，反映帳戶真實財富變化
自動扣除：從 Transactions CSV 自動識別 Wire Received 和 Journal 的今年淨流入，修正外部入金的影響
📌 需匯入 Transactions CSV 才能自動扣除入金，否則 YTD% 可能偏高
⚠ 呆滯料預警
觸發條件：周轉天數 > 30 天 且 未實現% 介於 -2% ~ +2%
含意：資金被壓住超過一個月，但股價橫盤不動，類似工廠的呆滯庫存
建議：評估是否減倉釋放資金投入高效標的
⚠️ 重建歷史快照
• 僅供第一次設定使用，日常操作不需要重建
• 重建會重新抓取歷史股價，若 Yahoo Finance 回傳資料不完整，可能導致歷史趨勢圖數值偏低
• 如果趨勢圖已有完整資料（993 筆以上），請勿再重建
• 重建前務必先備份
⚠️ 資料存放說明
• 所有資料存在瀏覽器 localStorage，不上傳任何伺服器
• 清除瀏覽器快取或資料會導致資料遺失，請定期備份
• 不同裝置的資料各自獨立，換裝置需透過備份 JSON 還原
⚠️ 股價延遲說明
• 即時股價來源：Finnhub 免費版，約 15 分鐘延遲
• 非盤中即時報價，僅供參考
• 歷史趨勢資料來源：Yahoo Finance，收盤價準確
